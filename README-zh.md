# Square 小方块

Square 小方块是一个轻量的 Obsidian 可视化打卡插件。

它适合那些你希望每天看得见、点得到、还能留在自己笔记库里的小事情：习惯、重复行动、次数记录、长期目标，以及一些简单但需要持续的日常项目。

Square 不想把打卡做成另一套很重的系统。它更像是 Obsidian 里的一个小方块面板：打开就能看到今天要做什么，点一下就能记录，后面需要回看时再看周视图、月视图和排行图表。

底层记录仍然保存在普通 Markdown 笔记里，所以它不会把你的数据从笔记库里拿走，也方便你阅读、迁移和手动检查。

## 为什么做 Square

很多打卡工具都能提醒你，但它们也会把记录带到另一套系统里。

Square 选择更轻的路线：

- 让今天要做的事直接可见
- 让记录足够快，快到日常真的愿意用
- 让底层数据继续留在 Markdown
- 只在需要回看时，再打开图表

它不是一个庞大的生活管理面板。它更适合帮你把一些小而重复的行动继续下去。

## 用起来是什么感觉

### 一个由小方块组成的主界面

每个项目都会变成主界面里的一个小方块。你可以从这里打卡、打开项目笔记、归档暂停的项目，也可以让今天的视图保持清爽。

### 一个快速记录弹窗

一次打卡可以写记录、选日期、调次数、填写固定字段，也可以带附件。简单项目可以接近“点一下就记完”的速度。

### 内置项目模板

Square 内置了常见打卡场景的模板，包括单次打卡、多次计数、目标追踪、热力图和趋势类项目。你不需要先维护一堆模板文件才能开始用。

### 需要回看时再打开总览

周视图、月视图和排行视图可以帮你回看近期进展。总览图表也可以放在侧栏里，作为一个比较轻的参考视图。

### 从月视图安排未来日期

对于未来日期，你可以提前安排那一天要显示哪些项目。如果某一天已经做了安排，到了那一天，主界面可以只显示被安排进去的项目，而不是把所有项目都铺出来。

## 现在可以做什么

- 在插件设置里创建和编辑 Square 项目。
- 用内置模板快速开始。
- 在可视化小方块主界面里打卡。
- 记录文字、日期、次数、固定字段和附件。
- 把项目数据保存在 Markdown 笔记里。
- 查看周视图、月视图和排行视图。
- 从月视图安排未来日期要出现的项目。
- 把暂停或结束的项目归档。
- 在 Obsidian 桌面端和移动端使用。

## 授权与版本

Square 小方块是专有软件。

未授权状态适合用少量项目试用核心流程。授权后可以解锁更完整的项目使用。

授权密钥通过 Obsidian 内置密钥管理保存，并在本地完成签名验证。正常本地使用不需要额外的服务器账号系统。

如需获取授权密钥，可以在小红书搜索 `焦应行`。

## 隐私与存储

- 项目记录会保存为你 Obsidian 笔记库里的 Markdown 笔记。
- 插件设置保存在本地 Obsidian 插件数据里。
- 授权密钥通过 Obsidian SecretStorage 保存。
- Square 目前不包含客户端遥测。
- 附件会根据插件流程和你的笔记库结构保存到本地库中。

不要公开你的 `.obsidian/plugins/square/data.json`。它可能包含本地设置、授权状态和历史配置数据。

## 安装

Square 通过 GitHub Releases 分发。

由于当前 Obsidian 官方插件商店规则要求上架插件开源，Square 在这个版本不会上架官方插件商店。

### 推荐方式：用 BRAT 安装

1. 先从 Obsidian 第三方插件里安装 `BRAT`。
2. 打开 BRAT 设置，选择 `Add Beta plugin`。
3. 填入这个仓库地址：

```text
https://github.com/jiaoyingxing/square
```

4. 让 BRAT 安装最新 release。
5. 在 Obsidian 第三方插件设置中启用 `Square`。

推荐用 BRAT，是因为它后续可以帮你从 GitHub Releases 更新，不用每次手动替换文件。

### 手动安装

1. 从最新 GitHub Release 下载 `main.js`、`manifest.json` 和 `styles.css`。
2. 在你的笔记库里创建这个目录：

```text
.obsidian/plugins/square/
```

3. 把下载的三个文件放进去。
4. 重启 Obsidian，或重新加载第三方插件。
5. 在 Obsidian 设置中启用 `Square`。

不要把 GitHub 自动生成的源码压缩包当成安装包。请使用 Release assets 里的安装文件。
